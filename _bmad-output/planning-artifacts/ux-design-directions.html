Design Direction Comparison

Six visual approaches for Antiphon's core screens. Each direction shows Dashboard, Workflow Detail, and Gate Review layouts. Click any card to view the full mockup.

A: Linear-Inspired Minimal

Clean single-column workflow list. Maximum whitespace. Focused, calm. Inspired by Linear's restrained density.

B: GitHub-Dense Table

Information-dense table layout. Compact rows, sortable columns. Maximum data visibility. GitHub's PR list aesthetic.

C: Split-Panel IDE

VS Code-style resizable panels. Stage tree on left, content on right. Power-user oriented. Maximum information density.

D: Pipeline-Forward

Stage progression dominates the page. Large visual pipeline at top, content below. Vercel/Azure DevOps inspired. Status-at-a-glance.

E: Card-Based Modular

Dashboard as a grid of rich cards. Each workflow is a self-contained card with inline status, mini-pipeline, and quick actions.

F: Conversation-First

Gate review centered on chat experience. Artifact as context panel on the side. Claude Code-style conversational flow.

Test Plan: Portfolio Rebalancing Alerts

v2

Acceptance Criteria

  • Alert triggers when portfolio drift exceeds configured threshold (default 5%)
  • Alert includes current allocation vs target allocation breakdown
  • Alert is delivered via configured channels (in-app, email, Teams)
  • Alert is suppressed during market close hours (configurable)

Test Scenarios

Happy Path

Given a portfolio with target allocation 60/40 stocks/bonds, when stock allocation drifts to 66% (6% drift, exceeding 5% threshold), then an alert is generated with the drift details and delivered to configured channels.

Edge Cases

  • Drift exactly at threshold (5.00%) — should trigger
  • Multiple simultaneous drifts across sub-portfolios
  • Alert during market close hours — suppressed and queued
  • Channel delivery failure — retry with exponential backoff

Integration Tests

Test the full pipeline from drift detection through AlertService to channel delivery:

// Verify alert creation and routing
var portfolio = CreateTestPortfolio(targetStocks: 0.60);
portfolio.SetCurrentAllocation(stocks: 0.66);
await _alertService.EvaluateDrift(portfolio);
Assert.Single(_channelMock.SentAlerts);
